# 前端数据加载问题修复说明

## 问题原因

后端 API 返回的数据格式是：
```json
{
  "status": 200,
  "success": true,
  "message": [实际数据数组]
}
```

而前端之前的代码没有正确处理后端的响应格式，导致无法获取数据。

## 修复内容

修改了 `frontend/api/axiosConfig.js` 的响应拦截器：

1. **正确解析后端响应**：
   - 从 `response.data.message` 字段提取实际数据
   - 将 `success === true` 作为成功判断条件

2. **错误处理优化**：
   - 统一处理后端返回的错误消息
   - 保持网络错误和HTTP错误的处理

## 现在的数据流程

```
后端: {status: 200, success: true, message: [数据]}
    ↓
axios 拦截器: 提取 message 字段
    ↓
前端 API: response.data 获取实际数据
    ↓
组件: 使用数据
```

## 测试步骤

1. **确保后端正在运行** (端口 8080)
2. **确保数据库有数据**：
   - 可以登录系统（登录成功后会有 token）
   - 确保有测试数据

3. **启动前端**：
   ```bash
   cd frontend
   npm run dev
   ```

4. **检查浏览器控制台**：
   - 按 F12 打开开发者工具
   - 查看 Network 标签
   - 刷新页面，查看 API 请求是否成功
   - 如果有错误，查看 Console 标签

## 可能的问题

### 1. 后端未启动
**现象**：前端显示"网络错误"  
**解决**：运行 `cd backend && mvn spring-boot:run`

### 2. 数据库没有数据
**现象**：API 返回成功，但列表为空  
**解决**：
- 可以通过其他方式（如数据库管理工具）添加数据
- 或者使用前端的添加功能添加测试数据

### 3. CORS 问题
**现象**：浏览器控制台显示 CORS 错误  
**解决**：检查后端 `CorsConfig.java` 是否正确配置（应该已经配置好了）

### 4. 401 未授权
**现象**：页面跳转到登录页  
**解决**：先登录系统，获取 token

## 验证数据已加载

1. 打开任意页面（如学生信息、宿舍楼信息）
2. 查看是否显示"加载中..."
3. 稍等片刻，应能看到数据表格
4. 如果显示"暂无数据"，说明 API 正常但数据库为空
